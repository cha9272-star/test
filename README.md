# 영업 Pipeline 재무 대시보드

영업 pipeline(엑셀)을 업로드하면 **월단위 매출 · 매출이익 · 운영자금 수지**를
자동 집계해 표와 차트로 보여주는 웹 앱입니다.

- **현금흐름은 수금월 기반**: 데이터의 `수금(월)`을 현금 유입 시점으로,
  `계산서(월)`을 매출 인식 시점으로 사용합니다. (수금월이 없으면 계산서월 + 입금 오프셋으로 보완)
- **데이터는 브라우저에서만 처리**됩니다. 서버로 전송되지 않아 영업데이터가 외부로 나가지 않습니다.

## 빠른 시작

```bash
npm install
npm run dev      # 개발 서버 (http://localhost:5173)
npm run build    # 정적 빌드 (dist/)
npm run preview  # 빌드 결과 미리보기
npm test         # 계산 엔진 + E2E 테스트
```

## 사용법

1. **업로드**: 영업 pipeline 엑셀(.xlsx)을 드래그&드롭.
2. **컬럼 매핑**: 데이터 시트와 헤더 행을 자동 인식하고, 각 열을 항목에 자동 매핑합니다.
   필요 시 드롭다운으로 수정하세요. 필수 항목은 **예상매출**, **매출인식월(계산서월)** 입니다.
   - `현금유입월(수금월)`을 지정하면 그 달에 현금 유입으로 잡습니다.
   - `원가(지출/매입)`를 지정하면 매출이익·현금유출 계산에 사용합니다. (없으면 기본 원가율로 추정)
3. **설정**: 기본 원가율, 월 고정 운영비, 시작 자금, 기준 연도, 분기 처리 방식, 가중(확률) 모드.
4. **대시보드**: 월별 매출/매출이익/현금수지 표 + 차트. 누적 운영자금이 마이너스가 되는 달은
   경고로 강조됩니다.
5. **내보내기**: 집계 결과를 엑셀로 다운로드.

## 월 표기 처리

다양한 실무 표기를 자동 인식합니다.

| 입력 예시 | 해석 |
|---|---|
| `2026-03`, `2026/3/15`, `2026년 3월`, `202603` | 2026-03 |
| `25/12월` | 2025-12 (2자리 연도) |
| `5월`, `1 월` | 기준 연도 적용 (예: 2026-05) |
| `26년 3Q`, `3분기` | 분기 처리 방식에 따라 배치 (기본: 마지막 달 → 9월) |
| `미정`, `상반기`, `하반기` | 월 확정 불가 → 집계 제외(별도 표시) |

## 계산 로직

- **매출 인식**: 계산서(월). 가중 모드면 `매출 × 수주확률`.
- **매출이익** = 매출 − 원가. 원가는 `지출(매입대)` 값, 없으면 `매출 × 기본 원가율`.
- **현금 유입** = 수금(월)에 매출액 입금. 수금월 없으면 계산서월 + 입금 오프셋.
- **현금 유출** = 지급월(없으면 계산서월 + 지급 오프셋)에 원가 지급.
- **운영자금 수지(순현금)** = 현금유입 − 현금유출 − 월 고정 운영비.
- **누적 자금** = 시작 자금 + 순현금 누계.

## 기술 스택

Vite + React + TypeScript · [SheetJS](https://sheetjs.com/) (엑셀 파싱/내보내기) · Recharts (차트)

핵심 계산 로직은 `src/lib/calc.ts`(순수 함수)에 있으며 `src/lib/*.test.ts`로 검증합니다.
